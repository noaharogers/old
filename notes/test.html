What is CSS?

I decided that if I was going to try and learn CSS than I should go ahead and start at the ground level. Today I found a great site called w3schools.com. This is what I learned about the meaning of CSS and why it was created.

  • CSS stands for Cascading Style Sheets.
  • Styles define how to display HTML elements.
  • The seperation of HTML from CSS makes it easier to maintain sites and share style sheets across numerous pages.
  • Styles were added to HTML 4.0 to solve a problem.
  • External Style Sheets can save a lot of work.
  • External Style Sheets are stored in CSS files.

WTF is a CSS Selector?

Within CSS there are items called selectors. To be honest, when first learning CSS this kind of blew me away. I did not think there would be any real way to distinguish between the three and I still do not completely understand them! Below are the main CSS selectors.

The element Selector

The element selector allows you to select and style elements within your HTML by adding styles to your CSS.

The id Selector

The id selector should be unique within a page, so you want to make sure you use this tag when you want to find a single, unique element. To find an element with a specific id, write #character, followed by the id of the element.

The class Selector

The class selector finds elements with the specific class. To find elements with a specific class, write .character followed by the class of the element.

Grouping them!

You can, and should group them when needed to save minimize code. When grouping, seperate each selector with a comma.

Oooohhh... Font Weights

I also learned about the font-weight; tag and how it can be used. While digging through other's code I would always see these tags, but never really understood exactly what it was and how it affected the font. I guess I may have over thought it a little bit. There are a few ways to use the font-weight; tag. Below are examples of the font-weight; tag.

  • font-weight; normal
  • font-weight; bold
  • font-weight; bolder
  • font-weight; lighter

With the font-weight; tag you can also specify how thin or thick you want your font to be using a numeric value. For reference, it is good to note that 400 is the same as normal, while 700 is the same as bold.

Inline and Block Elements

I have a few good friends who are extremely helpful when it comes to CSS and design. I regularly check my friend Adam Kaplan's website to view some of his projects for help in further understanding CSS and HTML. Below I have tried to replicate one of his earlier projects titled Inline and Block Elements.

display:inline;

Inline elements respect both left and right margins as well as padding, but not top and bottom. Inlines cannot have a width and height set, but allow other elements to sit to the left and right of them. Below is an example of three inline elements.

display: inline;
display: inline;
display: inline;

display:block;

Block elements respect all margins and force a line break after the actual block element. Below is an example of three block elements.

display:block;
display:block;
display:block;

display: inline-block;

Inline-Block elements respect all margins and padding as well as height and width parameters. These elements also allow other elements to sit to their left and right. Below is an example of three inline-block elements.

display: inline-block;
display: inline-block;
display: inline-block;
